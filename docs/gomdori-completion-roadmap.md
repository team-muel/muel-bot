# Gomdori 게임 완성 로드맵

작성 2026-06-10. Gomdori 마피아(Discord Activity)를 현재 상태에서 **출시 가능한 완성 게임**까지 끌고 가기 위한 전체 설계. 직업별 능력 한 개씩 쌓던 task 단위를 벗어나, 완성에 필요한 작업 전부를 마일스톤·워크스트림·수용기준으로 구조화한다.

정본/맥락: `docs/gomdori-mafia-design.md`, `docs/gomdori-w4-roles-design.md`, vault `Universes/BoW/Lore/Gomdori-마피아-규칙.md`, `src/gomdoriCodex.ts`(직업 v2 명세).

## 0. 현재 상태 스냅샷 (2026-06-10)

**구현됨**
- 페이즈 루프: `role_assign → 첫째 밤(무능력) → 낮 → 투표 → (찬반) → 처형/부결 → 의심 투표 → 밤 → 야간 해소 → …`. pg_cron + `phase-advance`.
- 승리 판정: 천사(악마 전멸) / 악마(카운트 패리티) / 중립(파스아 누적 전향 3) — `checkWinCondition`, 파스아 우선.
- 기본 로스터(16 명명 직업 + 파스아), "시민(무직)" 폐지. 진영 풀 배정.
- **변종 선택 배정**: 악마/조력자 슬롯 플레이어가 role_assign 단계에서 변종 선택(`match-select-role`), 미선택 랜덤 폴백.
- v2 고유능력(대부분): 봉인(세이카·팬텀·로건), 부활(미즐렛·헬렌), 변환(루나), 누진투표(엘런), 카운트(우노), 잔불대검(아서), 매료(루루). 팬텀 듀얼액션(처치+봉인).
- 이펙트 프리미티브: Kill/Heal/Protect/ModifyReceivedVote·Suspicion/ChangeFaction(중립화)/Silence/Corrupt/GrantCount/Charm.
- `/도감` 슬래시(진영별 직업 + 상세, 엔진 분리 데이터).
- UI: 로비, RoleAssign(변종 선택), Night(역할별 액션 블록 + 악마 채팅 + 보조액션 컴포넌트), Suspicion/Vote/Verdict, 관전 피드, Result(진영/중립 승리).

**미완 / 알려진 갭** (상세는 §10 인벤토리)
- 깊은 악마 4종 고유능력 미구현(말렌 빙의·혼/시체, 베스토 변신·히든, 대악마 낙인, 팬텀 악몽·일식).
- Edge Function 자동배포 없음 — 수동. **라이브 배포·동작 미검증**.
- Result reveal 이 DB role/faction 기준 → 전향/타락/변종/직업변환 흔적 미노출.
- 로비 UI 미흡(사용자 지적), 게임 설정 UI(중립 토글 등) 없음, 구성 미리보기 하드코딩.
- 다수 직업의 다단계 능력은 v2 시그니처 한 줄로 축약된 상태(원본 충실도 부족).
- 라이브 e2e QA 미수행. 자동 풀게임 시뮬레이션 테스트 없음(능력 단위 테스트만).

## 1. "완성"의 정의 (출시 기준)

게임이 **완성**되었다고 부르려면 아래 5축이 모두 합격선이어야 한다.

1. **플레이 가능(Playable)**: 5~12인이 처음부터 끝까지, 끊김·교착·크래시 없이 한 판을 마칠 수 있다(여러 페이즈, 능력 발동, 승리 판정까지).
2. **정합(Correct)**: 모든 배정·능력·판정이 정본과 일치하거나 의도된 v2 축약이며, 엣지케이스(동시성·재접속·중도이탈·교착)가 정의된 대로 처리된다.
3. **균형(Balanced)**: 진영 승률이 한쪽으로 붕괴하지 않고, 무한게임/즉승 같은 degenerate 상태가 없다(시뮬레이션 + 플레이테스트로 확인).
4. **운영(Operable)**: 배포가 재현 가능하고, 페이즈 진행 cron·env·시크릿이 정의되며, 장애 시 관측·복구가 가능하다.
5. **경험(Experience)**: 첫 진입부터 결과까지 UX가 매끄럽고(모바일 포함), 규칙을 모르는 사람도 온보딩되며, 정체성(비주얼·톤)이 일관된다.

## 2. 마일스톤 개요

| # | 마일스톤 | 목표 | 합격 게이트 |
|---|---|---|---|
| **M1** | 능력 완성 | 깊은 악마 4종 + 다단계 축약 보강 | 모든 직업이 정본/의도 v2대로 동작, 능력별 테스트 |
| **M2** | 루프 견고성 | 동시성·재접속·중도이탈·타임아웃·교착 처리 | 풀게임 e2e 시뮬레이션 통과, 엣지케이스 명세화 |
| **M3** | 배정·로비 | 게임 설정 UI + 로비 개편 + 구성 정확화 | 호스트가 옵션 정해 시작, 미리보기 정확 |
| **M4** | 공개·투명성 | 결과 reveal에 변환/변종/이력 반영 | 종료 시 모든 직업·변환 이력 정확 노출 |
| **M5** | 배포·운영 | edge 배포·migration·cron 자동화 + 라이브 검증 | 1버튼 배포, 라이브 풀게임 1판 성공 |
| **M6** | 검증 | 풀게임/밸런스 시뮬 + 플레이테스트 | 시뮬 스위트 + 플레이테스트 체크리스트 통과 |
| **M7** | UX·폴리시 | 비주얼/모션/모바일/접근성/카피/에러 | 디자인 합격선 + a11y 기본 |
| **M8** | 온보딩·디스코드 | 규칙/튜토리얼·결과요약·매칭·알림 | 신규 유저가 도움 없이 1판 |
| **M9** | 관측성 | 이벤트 로깅·게임 분석·에러 추적 | 판당 지표 수집, 장애 알림 |

권장 순서: **M5(배포·라이브 검증) 먼저** → M2(견고성) → M1(능력) → M4(reveal) → M3(로비) → M6(검증) → M7/M8 병행 → M9. 이유: 라이브에서 실제로 한 판 도는지 확인되지 않으면 나머지 작업의 토대가 흔들린다.

## 3. M1 — 능력 완성

깊은 악마 4종은 각자 다단계 상태기계라 직업당 새 프리미티브 + 상태가 필요하다. 정본은 vault 각 카드 + `gomdoriCodex.ts`의 `v2` 필드.

- **M1-1 팬텀 악몽·일식**
  - 악몽: 새 이펙트 `MarkNightmare`(대상 태그) + phase-advance 야간 해소에 "악몽→영면→다음 아침 탈락" 진행. 일식: `PhaseSwap`(다음 아침을 밤으로) + 팬텀 자기 소멸.
  - 수용: 악몽 지정 2밤 연속 → 탈락, 일식 1회로 아침 스킵. 테스트: 지연 탈락 타이밍.
- **M1-2 말렌 빙의·혼·시체**
  - 빙의: `Possess`(대상 그 밤 행동 봉인 + 그 밤 악마팀 카운트 가산). 혼/시체: 탈락자→혼 카운터→시체(투표/공격 보조). 혼령 방출 다단계.
  - 수용: 빙의 대상 행동 불가 + 카운트 전환, 혼 2개→시체 1.
- **M1-3 대악마 메피스토 낙인**
  - `Brand`(투표 대상 낙인) + 직업 삭제→비밀 재배정(새 천사 직업, 본인 미인지 상태). 사탄의 마(천사팀 투표가치 -1).
  - 수용: 낙인 대상이 다음 처형 무효화 또는 재배정. (가장 복잡 — 마지막 권장.)
- **M1-4 베스토 변신·히든 포지션**
  - `AltSelf`(밤마다 솔/하베스토 판정 토글: 조사 결과·투표가치 변동) + 히든 포지션(언급 기반 탈락) + 배후(조력자 효과 반전).
  - 수용: 변신 상태에 따라 조사 결과 달라짐, 언급 탈락 작동.
- **M1-5 다단계 보강(선택)**: 도르단 단서/사건의 전말, 하브레터스 소명 쿨다운, 로마즈 조사장, 라이너 백호 소환, 가인 약간의 위선/급습 등 — v2 축약을 원본 다단계로 끌어올리는 후속. 출시 필수 아님(균형 보고 결정).
- **M1-6 능력 상호작용 매트릭스**: 봉인↔보호↔부활↔변환↔처치 우선순위/충돌 표 정리(canon §17 "한 게임 한 주인공"). 충돌 시 악마 유리 규칙(canon §30) 코드화.

## 4. M2 — 게임 루프 견고성

능력보다 **먼저** 단단해야 출시 가능. 대부분 이미 부분 처리(GAME-1 동시성 가드 등)되어 있으나 명세·테스트가 빈다.

- **M2-1 동시성**: `phase-advance` 중복 호출(cron 겹침) 시 페이즈 이중전이 방지(현 claimedPhase 가드 검증) + 액션 upsert 경합.
- **M2-2 재접속/상태 복원**: 새로고침·재입장 시 내 역할/제출 액션/페이즈 타이머 복원(현 restoreAction 확장 — 변종 선택 중 새로고침 포함).
- **M2-3 중도이탈·강퇴·사망 관전**: 게임 중 leave/kick 시 카운트·승리 영향, 관전 모드 전환, 호스트 이탈 시 위임/중단.
- **M2-4 타임아웃·기권 기본값**: 각 페이즈 미제출 시 기본 동작(무투/미발동) 정의. role_assign 미선택 폴백(구현됨)·밤 액션 미제출·투표 미참여.
- **M2-5 교착 방지**: 동률 부결, 전원 사망, 능력만 남은 상태, 무한게임(라이너 카운트 등) 종료 보장. 최대 일수/강제 종료.
- **M2-6 페이즈 전이 정합**: 처형 후 승리체크, 야간 사망 후 승리체크, 첫째 밤 스킵 등 모든 전이의 승리판정 누락 점검.
- **M2-7 풀게임 e2e 시뮬레이션**: 엔진+페이즈 전이를 코드로 한 판 끝까지 돌리는 결정론적 시뮬(여러 시나리오: 천사승/악마승/중립승/교착). **이게 M2의 핵심 게이트.**

## 5. M3 — 배정·로비

- **M3-1 게임 설정 UI**: 호스트가 로비에서 옵션 설정 — 중립(파스아) 포함(`settings.includeNeutral`), 인원별 변형, (후속) 직업 풀 커스텀. 백엔드 게이트는 구현됨(토글 UI만).
- **M3-2 로비 개편**: 사용자 지적("UI 별로"). 준비/시작 흐름, 초대, 참가자 카드, 역할 안내(/도감 연계).
- **M3-3 구성 미리보기 정확화**: 현재 "악마 2 · 천사 N" 하드코딩 → 실제 분포(악마1+조력자1, 중립 여부, 천사 N) 반영.
- **M3-4 변종 선택 UX 보강**: 순차(악마→조력자→천사) 강조, 타이머, 선택 미리보기, 재선택 잠금.
- **M3-5 인원 경계**: 5~12 외 처리, 게임 중 인원 변동 시 분포 재계산 정책.

## 6. M4 — 공개·투명성 (Reveal)

현재 Result 가 DB role/faction 만 노출 → 게임 내 변환이 안 보임. canon §9: 종료 시 직업 변화(이전→변한 직업) 모두 공개.

- **M4-1 변환 이력 노출**: engine_state(currentRole/currentFaction)를 reveal 뷰로 plumbing — 전향자(원직업→파스아교세), 타락자(천사→악마팀), 변종(대악마/팬텀/…) 정확 표기.
- **M4-2 직업 변환 일반화**: 칼릭스(물려받기)·플로이·아델 등 후속 변환 직업도 이력 표기 가능한 구조.
- **M4-3 게임 요약**: 누가 누구를 처치/조사/전향했는지 핵심 사건 타임라인(관전 피드 확장 → 종료 리포트).

## 7. M5 — 배포·운영 (권장 최우선)

세션 내내 **edge 자동배포 없음 + 라이브 미검증**이 반복 지적됨. 여기를 먼저 닫아야 나머지가 실재한다.

- **M5-1 Edge 배포 파이프라인**: `match-*`·`phase-advance`·`match-select-role` 일괄 배포 스크립트/CI(merge 시 자동 또는 1커맨드). 현 `npm run functions:deploy` 확장.
- **M5-2 Migration 적용 게이트**: 머지된 migration 자동/반자동 적용 + 드리프트 0 확인(supabase-ops 패턴).
- **M5-3 phase-advance cron**: pg_cron 주기·`PHASE_ADVANCE_CRON_SECRET` 설정 검증, 누락 시 게임이 안 돈다.
- **M5-4 라이브 풀게임 검증**: 실제 Discord Activity 에서 5인 1판 끝까지(현재까지 미수행). **M5의 게이트.**
- **M5-5 환경/시크릿**: 봇 토큰·Supabase·게임 JWT·Render/Vercel 배포 경로 문서화 + 헬스체크.
- **M5-6 롤백**: migration·edge 롤백 절차.

## 8. M6 — 검증

- **M6-1 풀게임 시뮬 스위트**: M2-7 확장 — 직업 조합별·인원별 자동 플레이아웃(능력 상호작용 회귀).
- **M6-2 밸런스 시뮬**: N판 몬테카를로로 진영 승률·게임 길이 분포 → degenerate 탐지(무한게임/즉승). 능력 수치(봉인/부활/전향 임계 등) 튜닝 근거.
- **M6-3 플레이테스트 체크리스트**: 사람 5~12인 수동 QA 시나리오(각 직업 1회 이상 발동, 각 승리 경로 1회).
- **M6-4 회귀 게이트**: `test:gomdori`(현 6 스위트)에 풀게임·밸런스 추가, CI 필수화.

## 9. M7 / M8 / M9 — 경험·온보딩·관측

- **M7 UX·폴리시**: 비주얼 리프레시 후속(페이즈 앰비언스·토큰), 마이크로모션, 모바일/Activity 사이징, 접근성(키보드/aria/색대비), 카피·톤 일관, 에러/로딩 상태, (선택) 사운드.
- **M8 온보딩·디스코드**: 규칙/튜토리얼(첫 진입), 인게임 /도감 연계, 종료 후 채널 결과 요약 포스트, 매칭/큐(불특정 다수), 페이즈 알림. (canon §32 디스코드 채널 구조는 기획 보류 중 — 결정 필요.)
- **M9 관측성**: 게임 이벤트 로깅(이미 match_events) → 분석 대시보드, 에러 추적(Sentry류), 판당 지표(완주율·진영승률·평균 길이).

## 11. 미해결 결정(사용자 몫)

밸런스 정책은 "어지간하면 원문 따름"(2026-06-10). 그래도 명시 결정이 필요한 것:
